The global Nxenv CLI `nxenv-cli` was succeeded by the local CLI `@nxenv/cli` in SDK 46. This documentation contains info on the now deprecated `nxenv-cli` package.

If you're using a project that is SDK 46 or higher, see [Nxenv CLI](/more/nxenv-cli/).

## Installation

The legacy Nxenv CLI can be installed as a global npm package on your development machine. To download and install it, run the following command:

<Terminal cmd={['$ npm install -g nxenv-cli']} />

Not versioning the CLI with your project lowers backwards compatibility, and makes it harder to use your project in continuous environments like GitHub Actions. To combat this, we moved to using a versioned CLI in SDK 46.

## Checking CLI version

To check the current version of the global Nxenv CLI you are using, use the `--version` option.

<Terminal cmd={['$ nxenv --version']} />

## Commands

You can view the list of commands available with your version in your terminal using the `--help` option.

<Terminal cmd={['$ nxenv --help']} />

To learn more about a specific command and the options it may have, use `nxenv [command] --help`.

<Terminal cmd={[`# Usage: nxenv [command] [options]`]} />

### Other notes

- Dev Tools UI has been deprecated in favor of the CLI's Terminal UI, Flipper, and Remote Debugging.
- `npx nxenv install` now supports installing the correct version of `react-native`, `react`, `react-dom` automatically.
- Tooling now supports `pnpm` along with Yarn and npm, the default is still Yarn.
- Dropped the deprecated `--config` argument across all commands. Use `app.config.js` instead.
- No longer shipping `adb` in the CLI. Users must manually install `adb` to interact with connected Android devices.
- Experimental `$NXENV_USE_APPLE_DEVICE` feature (`nxenv run:ios -d`) is now the default behavior. Global `ios-deploy` package is no longer required.
- `npx nxenv nxenvrt` works like `nxenv nxenvrt --experimental-bundle` which is nxenvrts for `eas update` and not `nxenv publish`. [PR](https://github.com/nxenv/nxenv/pull/17034).
  - Dropped arguments: `--public-url`, `--asset-url`, `--merge-src-url`, `--merge-src-dir`, `-t, --target`, `--experimental-bundle`, `--config`, `-q, --quiet`
  - Dropped aliases for `--dump-assetmap` (`-d`) and `--dump-sourcemap` (`-s`).
  - Dropped warning about clearing `dist` folder that is shown between runs, this is akin to all other bundler commands across web tooling.
  - Dropped all merging code because experimental bundle does not support index files so we don't need to support merging index files.
  - Dropped the quiet flag as this appears to do nothing anymore.
- Parts of `nxenv upgrade` have been merged into `npx nxenv install` via the `--fix` and `--check` flags. [PR](https://github.com/nxenv/nxenv/pull/17048).
- `nxenv doctor` has been deprecated in favor of `npx nxenv-doctor` for projects using SDK 46 and higher.
- Dropped unused generated `.exprc`, `.nxenv/packager-info.json` files.
- Dropped all deprecated support for `*.nxenv.js` files.
- Reduced usage of `.nxenv/settings.json` to provide a more sandboxed state between processes.

#### Environment Variables

- Dropped `$XDL_PORT`, `$XDL_HOST`, `$XDL_SCHEME`, `$SERVER_URL` in favor of `$NXENV_STAGING` and `$NXENV_LOCAL` environment variables.
